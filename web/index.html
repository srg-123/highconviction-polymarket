Strategy: momentum_reversal (mom_fast)

Every ATP match on Polymarket is a binary market: one player's YES contract and the other's, priced to sum to ~$1. The strategy watches the overnight pre-match window — the 12–20 hours between market creation and match start — and fades overreactions in either direction.

Signal: if a player's YES price dropped 5+ cents over the last 5 hours → buy that contract, expecting it to recover. If it rose 5+ cents → buy the opposing player's (the one that got cheap), same reversion logic.

Why it works: pre-match pressure concentrates on one player at a time — news, social media, sharp money — and tends to overshoot. The strategy buys the contract that got hit, before the market corrects. Confirmed via Polymarket's startTime field: 308 of 309 signals fire before match start. In-match moves reflect real gameplay, not sentiment.

Stop-loss at 10¢. Optimal stake 15% compounding (out-of-sample selection only — use 2–5% live).

Schema

events id, title, series_id, sport, start_date, end_date, game_start_time One row per match. game_start_time is the actual Polymarket kickoff timestamp used to enforce the pre-match trading window.
markets id, event_id, question, yes_token_id, resolved_outcome, volume One row per binary outcome market (e.g. "Will Djokovic win?"). resolved_outcome: 1 = YES won, 0 = NO won.
price_history market_id, timestamp, price Hourly YES-price snapshots (0.0–1.0). Collected from clob.polymarket.com/prices-history. 1.5M+ rows.

API endpoints

GET /stats Database summary — event and market counts by sport, total price points.
GET /backtest Run any registered strategy against the dataset. Params: sport, strategy, entry_pct, trade_window, sizing, stop-loss, spread modeling.
GET /calibration Price bucket → actual win rate calibration table for any sport.
POST /strategies/generate Pass a plain-English strategy description; returns a Claude-generated Python predicate, AST-validated and registered into the live backtester.
